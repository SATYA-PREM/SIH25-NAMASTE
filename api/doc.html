Enhanced Medical Terminology API

Complete reference for the AYUSH-ICD-11 medical terminology microservice with FHIR R4 support, concept mapping, and unified search capabilities.

Overview

This enhanced medical terminology API bridges traditional AYUSH systems (Ayurveda, Siddha, Unani) with modern ICD-11 classifications. It provides unified search, concept mapping, FHIR R4 compliance, and extensible terminology management.

  • Multi-system search: Ayurveda, Siddha, Unani, ICD-11, and doctor-added terms
  • Concept mapping: Automated cross-system term relationships
  • FHIR R4 support: ValueSet $expand, ConceptMap $translate operations
  • Doctor terminology: Extensible database for custom medical terms
  • ICD-11 integration: Automated mapping and lookup capabilities
  • Export functionality: FHIR Bundle generation for interoperability

System Architecture

The system consists of multiple SQLite databases with unified access patterns:

  • Legacy Databases: Ayurveda.db, Siddha.db, Unani.db, ICD11.db
  • Doctor Database: doctorterminology.db (extensible terminology)
  • Mapping Database: conceptmappings.db (cross-system relationships)
  • Search Engine: Multi-threaded unified search with relevance ranking
  • FHIR Layer: Standards-compliant FHIR R4 operations

CLI Commands

Use the command-line interface for debugging, testing, and administration without HTTP requests.

Server Mode

Start the API server with various configuration options:

# Basic server start (localhost:5000)
python fixed_api_server.py

# Server with custom host and port
python fixed_api_server.py --host 0.0.0.0 --port 8080

# Server on specific interface
python fixed_api_server.py --host 192.168.1.100 --port 5000

Debug Mode

Inspect database structures, tables, and sample data:

# Debug all databases
python fixed_api_server.py --mode debug

# Debug specific database
python fixed_api_server.py --mode debug --db ayurveda
python fixed_api_server.py --mode debug --db icd11
python fixed_api_server.py --mode debug --db siddha
python fixed_api_server.py --mode debug --db unani
python fixed_api_server.py --mode debug --db doctor

Search Mode

Perform searches directly from command line:

# Search across all databases
python fixed_api_server.py --mode search "diabetes"

# Search specific database
python fixed_api_server.py --mode search "fever" --db ayurveda

# Search with system filter
python fixed_api_server.py --mode search "pain" --system "Ayurveda" --limit 10

# Search doctor terminology
python fixed_api_server.py --mode search "hypertension" --db doctor

# Search with result limit
python fixed_api_server.py --mode search "respiratory" --limit 20

Unified Search

# Search across all databases
curl "http://localhost:5000/api/search?term=diabetes"

# Search specific database
curl "http://localhost:5000/api/search?term=fever&database=ayurveda"

# Search with system filter
curl "http://localhost:5000/api/search?term=pain&system=Ayurveda"

# Search with limits
curl "http://localhost:5000/api/search?term=heart&total_limit=20"
curl "http://localhost:5000/api/search?term=respiratory&database=ayurveda&limit_per_db=5"

# Advanced search parameters
curl "http://localhost:5000/api/search?term=madhumeha&database=ayurveda&system=Ayurveda&total_limit=10"

Add Terminology

# Add new doctor terminology (POST request)
curl -X POST "http://localhost:5000/api/add-terminology" \
  -H "Content-Type: application/json" \
  -d '{
    "doctor_name": "Dr. Smith",
    "system": "Modern Medicine",
    "code": "HYP001",
    "term": "Essential Hypertension",
    "short_definition": "High blood pressure without known cause",
    "long_definition": "Essential hypertension is high blood pressure that does not have a known secondary cause...",
    "reference": "Medical textbook reference",
    "ontology_branches": "Cardiovascular System"
  }'

FHIR R4 Endpoints

Standards-compliant FHIR operations for interoperability.

ValueSet Operations

# Expand ValueSet - get all concepts
curl "http://localhost:5000/fhir/ValueSet/\$expand?identifier=ayurveda-terms"
curl "http://localhost:5000/fhir/ValueSet/\$expand?identifier=all-traditional"
curl "http://localhost:5000/fhir/ValueSet/\$expand?identifier=icd11-terms"

# Validate code in ValueSet
curl -X POST "http://localhost:5000/fhir/ValueSet/\$validate-code" \
  -H "Content-Type: application/fhir+json" \
  -d '{
    "resourceType": "Parameters",
    "parameter": [
      {
        "name": "code",
        "valueCode": "C001"
      },
      {
        "name": "url",
        "valueUri": "http://localhost:5000/fhir/ValueSet/ayurveda-terms"
      }
    ]
  }'

ConceptMap Operations

# Translate concept between systems
curl -X POST "http://localhost:5000/fhir/ConceptMap/\$translate" \
  -H "Content-Type: application/fhir+json" \
  -d '{
    "resourceType": "Parameters",
    "parameter": [
      {
        "name": "code",
        "valueCode": "NAMC"
      },
      {
        "name": "system",
        "valueUri": "http://localhost:5000/fhir/CodeSystem/ayurveda"
      },
      {
        "name": "targetsystem",
        "valueUri": "http://localhost:5000/fhir/CodeSystem/icd11"
      }
    ]
  }'

Concept Mapping Endpoints

Manage cross-system terminology relationships and mappings.

Add Concept Mapping

# Add new concept mapping
curl -X POST "http://localhost:5000/api/add-mapping" \
  -H "Content-Type: application/json" \
  -d '{
    "source_system": "ayurveda",
    "source_code": "AYU001",
    "source_term": "Prameha",
    "target_system": "icd11",
    "target_code": "5A10",
    "target_term": "Diabetes mellitus",
    "relationship_type": "equivalent",
    "confidence_score": 0.95
  }'

Get Concept Mappings

# Get mappings for specific term
curl "http://localhost:5000/api/mappings?source_system=ayurveda&source_code=AYU001"

# Get mappings with target system filter
curl "http://localhost:5000/api/mappings?source_system=ayurveda&source_code=AYU001&target_system=icd11"

# ICD-11 lookup for term mapping
curl -X POST "http://localhost:5000/api/icd11-lookup" \
  -H "Content-Type: application/json" \
  -d '{
    "term": "Madhumeha",
    "system": "Ayurveda"
  }'

Mapping Statistics

# Get concept mapping statistics
curl "http://localhost:5000/api/mapping-stats"

# Get doctor terminology statistics
curl "http://localhost:5000/api/doctor-stats"

Troubleshooting Guide

Common issues and systematic debugging approach.

Step 1: Verify Server Status

# Check if server is running
curl "http://localhost:5000/api/health"

# Expected response
{
  "status": "healthy",
  "message": "Enhanced Medical Database API with Concept Mapping is running",
  "available_databases": 4,
  "total_databases": 4,
  "doctor_database": "available",
  "concept_mapping_database": "available"
}

Step 2: Database Structure Verification

# Debug database structure
curl "http://localhost:5000/api/debug/ayurveda"

# Check all databases
curl "http://localhost:5000/api/databases"

# CLI method
python fixed_api_server.py --mode debug --db ayurveda

Common Issues

# Issue: Connection refused
# Solution: Start the server
python fixed_api_server.py --host 0.0.0.0 --port 5000

# Issue: No search results for known terms
# Solution: Check database structure and column names
curl "http://localhost:5000/api/debug/ayurveda"

# Issue: ICD-11 mapping showing N/A
# Solution: Verify concept mappings database
curl "http://localhost:5000/api/mapping-stats"

# Issue: FHIR endpoints not working
# Solution: Check FHIR parameter format
curl -X POST "http://localhost:5000/fhir/ValueSet/\$validate-code" \
  -H "Content-Type: application/fhir+json" \
  -d '{"resourceType": "Parameters", "parameter": [{"name": "code", "valueCode": "test"}]}'
