Complete reference: how the API works, CLI commands, server options, curl examples and a step-by-step troubleshooting workflow so anyone can get the service running quickly.

Overview

This demo-ready terminology microservice bridges AYUSH NAMASTE concepts with WHO ICD-11 TM2 via FHIR R4 patterns. It ships with seeded values so you can search concepts, review dual-code mappings, expand ValueSets, and validate FHIR Bundles without external credentials.

  • Dual coding: NAMASTE concepts with suggested ICD-11 matches
  • FHIR operations: ValueSet $expand and ConceptMap $translate
  • CSV ingestion for NAMASTE concepts
  • Demo-first: run immediately with seeded data

CLI Debugging Commands

Use the bundled CLI (the same script that runs the server) to inspect databases, run quick searches, and diagnose issues without firing HTTP requests.

1. Debug All Databases

Show structure of all databases (tables, columns, sample data):

python fixed_api_server.py --mode debug

2. Debug Specific Database

Inspect individual databases:

python fixed_api_server.py --mode debug --db ayurveda
python fixed_api_server.py --mode debug --db icd11
python fixed_api_server.py --mode debug --db siddha
python fixed_api_server.py --mode debug --db unani

3. CLI Search Commands

Run searches from the CLI — useful for scripted checks and smoke tests.

# Search across all databases
python fixed_api_server.py --mode search "diabetes"

# Search in specific database
python fixed_api_server.py --mode search "fever" --db ayurveda

# Search with limit
python fixed_api_server.py --mode search "heart" --db ayurveda --limit 10

# Search with system filter
python fixed_api_server.py --mode search "pain" --system "Ayurveda" --limit 5

API Server Host Configuration

Run the server on localhost (development) or bind to all interfaces for network access.

Run on Specific Host (Localhost Only)

# Default - accessible only from same machine
python fixed_api_server.py --host 127.0.0.1 --port 5000

# Alternative localhost syntax
python fixed_api_server.py --host localhost --port 5000

Run on All Hosts (Network Access)

# Accessible from any machine on network
python fixed_api_server.py --host 0.0.0.0 --port 5000

# Custom port on all hosts
python fixed_api_server.py --host 0.0.0.0 --port 8080

Custom Host and Port Options

# Specific IP address
python fixed_api_server.py --host 192.168.1.100 --port 5000

# Different port
python fixed_api_server.py --host 0.0.0.0 --port 3000

API Debug Endpoints (curl)

Use curl to call debug endpoints — useful when the server is running on another machine or when you want to add checks to CI.

Using curl for Localhost

# Health check
curl "http://localhost:5000/api/health"

# Debug specific database
curl "http://localhost:5000/api/debug/ayurveda"
curl "http://localhost:5000/api/debug/icd11"
curl "http://localhost:5000/api/debug/siddha"
curl "http://localhost:5000/api/debug/unani"

# List all databases
curl "http://localhost:5000/api/databases"

# Search for term
curl "http://localhost:5000/api/search?term=diabetes&database=ayurveda"

Using curl for Network Access (0.0.0.0)

# Replace YOUR_SERVER_IP with actual IP
curl "http://YOUR_SERVER_IP:5000/api/health"
curl "http://YOUR_SERVER_IP:5000/api/debug/ayurveda"
curl "http://YOUR_SERVER_IP:5000/api/search?term=diabetes"

# Example with actual IP
curl "http://192.168.1.100:5000/api/debug/ayurveda"

Complete Troubleshooting Workflow

Step-by-step flow from starting the server to verifying data and search functionality.

Step 1: Start API Server

# For local development
python fixed_api_server.py --host 127.0.0.1 --port 5000

# For network access
python fixed_api_server.py --host 0.0.0.0 --port 5000

Step 2: Verify Server is Running

# Test API health
curl "http://localhost:5000/api/health"

# Or for network access
curl "http://YOUR_IP:5000/api/health"

Step 3: Debug Database Structure

# CLI method - debug all databases
python fixed_api_server.py --mode debug

# API method - check specific database
curl "http://localhost:5000/api/debug/ayurveda"

Step 4: Test Search Functionality

# CLI search
python fixed_api_server.py --mode search "diabetes" --db ayurveda

# API search
curl "http://localhost:5000/api/search?term=diabetes&database=ayurveda"

Common Issues & Solutions

Issue 1: "Connection refused"

Solution: Make sure the API server is running:

python fixed_api_server.py

Issue 2: "No results found" for known terms

Solution: Run the debug command to inspect the actual structure and sample rows:

curl "http://localhost:5000/api/debug/ayurveda"

Issue 3: Wrong database name

Solution: List available databases first:

curl "http://localhost:5000/api/databases"
