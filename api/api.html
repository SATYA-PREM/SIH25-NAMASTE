Overview

The Medical Terminology API provides access to a comprehensive database of medical terms from traditional systems (Ayurveda, Siddha, Unani) and modern medical standards (ICD-11). This RESTful API supports healthcare applications, research platforms, and medical information systems.

Key Features

  • Cross-System Search: Search across multiple medical terminology systems
  • Doctor Contributions: Healthcare professionals can add verified terms
  • CSV Data Import: Bulk upload medical terminology datasets
  • Real-time Results: Fast search with intelligent matching
  • Structured Data: Consistent JSON responses with rich metadata

Base URL

https://api.medunify.com/v1

Development: http://localhost:5000

Authentication

Currently, the API is open for demo purposes. In production, API keys will be required: 

Authorization: Bearer YOUR_API_KEY

Available Endpoints

Method Endpoint Description
GET /api/health Check API health status
GET /api/search Search medical terminology
POST /api/add-terminology Add new medical term
GET /api/databases List available databases
GET /api/debug/{database} Debug database structure

Search API

GET /api/search

Query Parameters

Parameter Type Required Description
term string Yes Medical term to search for
database string No Specific database: ayurveda, icd11, siddha, unani, doctor
system string No Filter by medical system
total_limit integer No Maximum results to return (default: 300)
limit_per_db integer No Results per database when searching all

Example Request

GET /api/search?term=diabetes&database=ayurveda&total_limit=10

Example Response

Add New Terminology

POST /api/add-terminology

Request Body

Field Type Required Description
doctor_name string Yes Name of contributing doctor
system string Yes Medical system (e.g., Ayurveda)
term string Yes Medical term
short_definition string Yes Brief definition
long_definition string No Detailed explanation
code string No Medical code identifier
reference string No Source reference
ontology_branches string No Classification categories

Example Request

POST /api/add-terminology
Content-Type: application/json

{
  "doctor_name": "Dr. Rajesh Kumar",
  "system": "Ayurveda",
  "term": "Pittaja Jwara",
  "short_definition": "Fever caused by pitta dosha imbalance",
  "long_definition": "A type of fever in Ayurveda characterized by...",
  "code": "PJ001",
  "reference": "Charaka Samhita",
  "ontology_branches": "Fever Types, Pitta Disorders"
}

Database Information

GET /api/databases

Available Databases

  • ayurveda: Traditional Indian medical system terminology
  • siddha: Traditional Tamil medical system terms
  • unani: Greco-Arabic medical system vocabulary
  • icd11: WHO International Classification of Diseases
  • doctor: User-contributed medical terms

Debug Tools

GET /api/debug/{database}

Get detailed information about database structure, including tables, columns, and sample data.

Example

GET /api/debug/ayurveda

Usage Examples

JavaScript (Fetch API)

// Search for medical terms
const response = await fetch('/api/search?term=fever&database=ayurveda');
const data = await response.json();
console.log(data.data);

// Add new terminology
const newTerm = {
  doctor_name: "Dr. Smith",
  system: "Ayurveda",
  term: "Vata Dosha",
  short_definition: "One of three fundamental doshas"
};

const addResponse = await fetch('/api/add-terminology', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(newTerm)
});

cURL Examples

# Search across all databases
curl "http://localhost:5000/api/search?term=diabetes&total_limit=5"

# Search specific database
curl "http://localhost:5000/api/search?term=fever&database=ayurveda"

# Add new term
curl -X POST "http://localhost:5000/api/add-terminology" \
  -H "Content-Type: application/json" \
  -d '{
    "doctor_name": "Dr. Patel",
    "system": "Ayurveda",
    "term": "Kapha Dosha",
    "short_definition": "One of three fundamental energies"
  }'

Python Requests

import requests

# Search example
response = requests.get('http://localhost:5000/api/search', {
    'term': 'diabetes',
    'database': 'ayurveda',
    'total_limit': 10
})

results = response.json()
print(f"Found {results['count']} results")

# Add term example
new_term = {
    'doctor_name': 'Dr. Sharma',
    'system': 'Ayurveda',
    'term': 'Pitta Dosha',
    'short_definition': 'Fire and water element combination'
}

add_response = requests.post(
    'http://localhost:5000/api/add-terminology',
    json=new_term
)
