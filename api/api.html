Overview

The Enhanced Medical Terminology API provides comprehensive access to traditional and modern medical terminology systems. It bridges AYUSH traditional medicine (Ayurveda, Siddha, Unani) with international standards like ICD-11, offering advanced search capabilities, concept mapping, and FHIR R4 compliance.

Key Features

  • Multi-System Search: Search across 5 medical terminology systems simultaneously
  • Doctor Contributions: Healthcare professionals can add verified medical terms
  • Concept Mapping: Cross-system terminology relationships with confidence scoring
  • ICD-11 Integration: Automated lookup and mapping to WHO ICD-11 codes
  • FHIR R4 Support: ValueSet $expand and ConceptMap $translate operations
  • Real-time Results: Multi-threaded search with intelligent relevance ranking
  • Export Capabilities: Generate FHIR Bundles for interoperability

Unified Search API

GET /api/search

The search endpoint provides intelligent, multi-threaded search across all medical terminology databases with relevance ranking and system filtering.

Query Parameters

Parameter Type Required Description Example
term string Yes Medical term to search for (case-insensitive) diabetes, fever, madhumeha
database string No Specific database to search ayurveda, icd11, siddha, unani, doctor
system string No Filter by medical system Ayurveda, Siddha, Unani
total_limit integer No Maximum total results (default: 300) 50, 100, 200
limit_per_db integer No Results per database when searching all 10, 25, 50

Example Requests

# Search across all databases
GET /api/search?term=diabetes&total_limit=20

# Search specific database
GET /api/search?term=madhumeha&database=ayurveda

# System-filtered search
GET /api/search?term=fever&system=Ayurveda&total_limit=15

# Multi-database with limits
GET /api/search?term=respiratory&limit_per_db=5&total_limit=25

Response Format

      "source_database": "ayurveda",

Add New Terminology

POST /api/add-terminology

Healthcare professionals can contribute verified medical terms to the doctor terminology database.

Request Body

Field Type Required Description
doctor_name string Yes Name of contributing doctor
system string Yes Medical system (e.g., Ayurveda, Modern Medicine)
term string Yes Medical term or condition name
short_definition string Yes Brief, concise definition
long_definition string No Detailed explanation and context
code string No Medical code identifier
reference string No Source reference or textbook citation
ontology_branches string No Classification categories or branches

Example Request

POST /api/add-terminology
Content-Type: application/json

{
  "doctor_name": "Dr. Rajesh Kumar",
  "system": "Ayurveda",
  "term": "Pittaja Jwara",
  "short_definition": "Fever caused by pitta dosha imbalance",
  "long_definition": "A type of fever in Ayurveda characterized by high temperature, burning sensation, and associated pitta symptoms like acidity and irritability.",
  "code": "PJ001",
  "reference": "Charaka Samhita, Nidana Sthana",
  "ontology_branches": "Fever Types, Pitta Disorders, Ayurvedic Pathology"
}

Concept Mapping

Manage cross-system terminology relationships and mappings between different medical systems.

Add Concept Mapping

POST /api/add-mapping

Request Body

Field Type Required Description
source_system string Yes Source system (ayurveda, siddha, unani)
source_code string Yes Source system code
source_term string Yes Source system term
target_system string Yes Target system (usually icd11)
target_code string Yes Target system code
target_term string Yes Target system term
relationship_type string No equivalent, related-to, broader-than (default: equivalent)
confidence_score float No Confidence level 0.0-1.0 (default: 1.0)

Get Concept Mappings

GET /api/mappings

Query Parameters

# Get mappings for specific term
GET /api/mappings?source_system=ayurveda&source_code=NAMC

# Get mappings with target system filter
GET /api/mappings?source_system=ayurveda&source_code=NAMC&target_system=icd11

ICD-11 Lookup New

POST /api/icd11-lookup

Automated lookup of ICD-11 codes and terms based on traditional medical terminology with confidence scoring.

Request Body

Field Type Required Description
term string Yes Medical term to find ICD-11 mapping for
system string No Source medical system for context

Example Request

POST /api/icd11-lookup
Content-Type: application/json

{
  "term": "Madhumeha",
  "system": "Ayurveda"
}

Response

200 OK 
{
  "code": "5A10",
  "term": "Diabetes mellitus",
  "system": "http://id.who.int/icd/release/11/mms",
  "confidence": 0.95
}

Complete Usage Examples

JavaScript (Fetch API)

// Search for medical terms with ICD-11 mapping
async function searchTerms(term, database = null) {
  const params = new URLSearchParams({ term });
  if (database) params.append('database', database);
  
  const response = await fetch(`/api/search?${params}`);
  const data = await response.json();
  
  // Display results with ICD-11 mappings
  data.combined_results.forEach(result => {
    console.log(`${result.term} -> ICD-11: ${result.icd11_code} (${result.icd11_term})`);
  });
}

// Add new terminology with doctor verification
async function addTerm(termData) {
  const response = await fetch('/api/add-terminology', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(termData)
  });
  
  if (response.ok) {
    const result = await response.json();
    console.log('Term added successfully:', result.id);
  }
}

// Get ICD-11 mapping for traditional term
async function getICD11Mapping(term, system) {
  const response = await fetch('/api/icd11-lookup', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ term, system })
  });
  
  const mapping = await response.json();
  return mapping;
}

Python Requests

import requests
import json

BASE_URL = "http://localhost:5000"

# Search with ICD-11 mapping
def search_with_icd11(term, database=None):
    params = {'term': term, 'total_limit': 20}
    if database:
        params['database'] = database
    
    response = requests.get(f"{BASE_URL}/api/search", params=params)
    results = response.json()
    
    for result in results['combined_results']:
        print(f"{result['term']} -> ICD-11: {result.get('icd11_code', 'N/A')}")
    
    return results

# Add concept mapping
def add_concept_mapping(source_sys, source_code, source_term, 
                       target_sys, target_code, target_term, confidence=1.0):
    mapping_data = {
        'source_system': source_sys,
        'source_code': source_code,
        'source_term': source_term,
        'target_system': target_sys,
        'target_code': target_code,
        'target_term': target_term,
        'relationship_type': 'equivalent',
        'confidence_score': confidence
    }
    
    response = requests.post(f"{BASE_URL}/api/add-mapping", json=mapping_data)
    return response.json()

# Get statistics
def get_system_stats():
    doctor_stats = requests.get(f"{BASE_URL}/api/doctor-stats").json()
    mapping_stats = requests.get(f"{BASE_URL}/api/mapping-stats").json()
    
    print(f"Doctor terms: {doctor_stats['total_terminology']}")
    print(f"Total mappings: {mapping_stats['total_mappings']}")

# Example usage
search_with_icd11("diabetes", "ayurveda")
get_system_stats()
